# ETL аналитический пайплайн (исправленная версия)

## Обзор

Проект реализует полный аналитический пайплайн обработки данных:

PostgreSQL → ETL → MinIO (Parquet, партиционирование) → Витрины данных → Apache Superset

Цель — подготовка сырых данных для аналитики и BI-дашбордов.

---

## Архитектура

- PostgreSQL — источник данных (скважины, добыча, телеметрия)
- ETL (Python / Pandas) — извлечение и преобразование данных
- MinIO (S3 хранилище) — хранение parquet-файлов с партициями
- Витрины данных — аналитические таблицы (mart_production, mart_failures, mart_logistics)
- Apache Superset — слой визуализации

---

## Поток данных

PostgreSQL  
→ ETL-скрипты  
→ MinIO (разбиение по year/month/day)  
→ Витрины данных  
→ Superset дашборды  

---

## Дашборд (Superset)

Дашборд содержит:

### KPI карточки
- Общая добыча
- Средний дебит
- Среднее давление
- Процент простоя

### Графики
- Добыча по времени (линейный график)
- Топ скважин (столбчатый график)
- Давление vs добыча (heatmap)
- Анализ простоев
