# Архитектура пайплайна данных

## Общая схема

PostgreSQL
    ↓
ETL (Python / Pandas)
    ↓
MinIO (Parquet, партиционирование по датам)
    ↓
Витрины данных
    ├── mart_production
    ├── mart_failures
    ├── mart_logistics
    ↓
Apache Superset
    ↓
Интерактивный дашборд

---

## Ключевые решения

- Использование Parquet для аналитической эффективности
- Партиционирование по времени (год/месяц/день)
- Разделение на витрины для разных бизнес-задач
- Superset как BI-инструмент

---

## Итог

Архитектура позволяет масштабировать систему и использовать данные как для аналитики, так и для ML-задач.
